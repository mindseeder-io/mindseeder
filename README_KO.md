# MindSeeder 초보자용 가이드

이 문서는 **프로그래밍 입문자**가 현재 프로젝트를 이해하고, 수정한 내용을 실제로 적용(배포)하는 데 도움을 주기 위한 단계별 안내입니다.

## 1) PR(Pull Request)이란?

PR은 쉽게 말해, 

1. 내가 코드를 고친 뒤,
2. "이 변경을 메인 코드에 합쳐주세요"라고 팀에 요청하고,
3. 리뷰를 거쳐 반영하는 절차

입니다.

즉, 단순 저장이 아니라 **변경 이유 + 내용 + 검증 결과를 함께 공유하는 공식 요청**입니다.

---

## 2) 지금 코드가 어디서 바뀌었는지 확인하기

터미널에서 프로젝트 폴더로 이동한 뒤 아래 명령을 사용하세요.

```bash
git log --oneline -n 5
git show --name-only
```

- `git log`: 최근 커밋(변경 기록) 목록 확인
- `git show`: 가장 최근 커밋의 실제 변경 파일 확인

---

## 3) 로컬(내 컴퓨터)에서 바로 실행해보기

정적 HTML 사이트는 별도 빌드 없이 간단히 확인할 수 있습니다.

```bash
python -m http.server 4173
```

브라우저에서 아래 주소 접속:

- `http://localhost:4173/index.html`

법률 페이지 확인:

- `http://localhost:4173/privacy.html`
- `http://localhost:4173/terms.html`

종료는 터미널에서 `Ctrl + C`.

---

## 4) 수정 내용을 실제 서비스에 적용하는 기본 흐름

배포 방식마다 세부는 다르지만, 큰 흐름은 같습니다.

1. 변경 파일 수정
2. 로컬에서 실행해서 확인
3. Git에 저장(commit)
4. 원격 저장소에 올림(push)
5. PR 생성
6. 리뷰 후 main 브랜치에 병합(merge)
7. 배포 자동/수동 반영 확인

예시 명령:

```bash
git add .
git commit -m "Update landing and legal pages"
git push origin <내-브랜치-이름>
```

그다음 GitHub에서 **Compare & pull request** 버튼을 눌러 PR을 생성합니다.

---

## 5) 초보자 체크리스트 (실수 방지)

- [ ] 링크가 잘 열리는가? (`privacy.html`, `terms.html`, `mailto:`)
- [ ] 모바일 화면에서 글자가 너무 작지 않은가?
- [ ] 오탈자/날짜가 맞는가?
- [ ] 커밋 메시지가 "무엇을 왜 바꿨는지" 드러나는가?

---

## 6) 용어 짧은 사전

- **commit**: 코드 변경을 한 단위로 저장한 기록
- **branch**: 메인 코드와 분리된 작업 공간
- **PR**: 작업 브랜치 변경을 메인에 합쳐달라고 요청하는 절차
- **merge**: PR이 승인된 후 변경을 메인 코드에 합치는 작업
